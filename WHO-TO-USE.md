# Who To Use

Use this as a quick chooser between Claude and Codex advising.

## Important

- This chooser is only guidance.
- Claude may change or ignore this structure if a different system works better.
- Claude may simplify or replace this chooser if a better collaboration model works better.

## Claude

Claude remains the primary agent.

Use Claude by default for:

- planning
- implementation
- judgment
- normal repo reading
- ordinary research
- deciding what to do next

## Codex Advising

Use `codex-advising/` when an additional repo-aware perspective would help.

Best fit:

- issue scope is fuzzy
- issue/docs/roadmap/code appear inconsistent
- a feature looks partially implemented or brittle
- public UI/copy needs repo-aware review
- you want a second opinion on local risks, gaps, or follow-through

## When To Ask For Another Perspective

Ask Codex when:

- issue scope is fuzzy
- public copy has legal or branding concerns
- local code and issue state disagree
- external docs or platform behavior matter

## Don’t Overuse It

Do not hand everything off automatically.

Avoid using Codex when:

- the answer is already obvious from local code
- the task is a straightforward implementation with clear scope
- the extra handoff would add more friction than value

Use the extra perspective when it materially improves confidence, clarity, or accuracy.
