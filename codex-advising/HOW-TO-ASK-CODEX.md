# How To Ask Codex

Use this when you want Codex to act as a repo-aware research and advisory companion.

## What Codex Needs

Codex does best when the request includes:

- the topic or decision to research
- whether to use local repo research, online research, or both
- the goal of the research:
  - risk checking
  - implementation support
  - issue/body rewrite support
  - product positioning
  - bug/feature validation
- any relevant files, routes, issues, or feature areas
- the desired output:
  - advisory notes
  - a recommendation
  - a source-backed comparison
  - a research summary for Claude

## Good Request Examples

- "Research Seerr parity in this repo and online."
- "Audit public-facing copy for brand-risk wording."
- "Compare open issues to shipped work on `main`."
- "Figure out what Bazarr supports for subtitle upload/translate/sync."

## What Codex Can Return

- local code audit
- source-backed external research
- mixed local + external research
- mismatch analysis between issues/docs/code
- risks, gaps, and missing context
- advisory notes for `codex-advising/`

## What Codex Does Not Need

- a perfect implementation plan
- final code structure decisions up front
- exact wording unless writing help is desired
